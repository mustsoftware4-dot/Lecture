# Лекц №6 "Төвөгтэй байдлыг бууруулах, Дизайныг дотоод шинж чанар "

## Сэдэв :

    - Программын төвөгтэй байдал (Software Complexity) гэж юу вэ?
    - Хэрхэн үүсдэг вэ? 
    - Төвөгтэй байдал нэмэгдэх ямар ямар шалтгаанууд байдаг вэ?

## Тодорхойлолт 

    - Программ эхэндээ жижигхэн ойлгоход амархан байдаг. Гэхдээ шинэ функц нэмж, алдаагаа засаж, программист хамтарч ажиллахад кодууд дахин дахин өөрчлөгдөнө. Үүний үр дүнд эмх замбараагүй байдал үүсэхийг программын төвөгтэй байдал буюу (Software Complexity) гэж нэрлэдэг.

   ###  Төвөгтэй байдал хэрхэн үүсдэг вэ?

   - Код дотор хэт олон нөхцөл шалгах(if, else, case)болон давталтууд бие бие дотроо орсон тохиолдолд системийн ажиллагааг хянахад хэцүү болдог.

   - Нэг код нөгөөгөө кодоосоо хэт хамааралтай байх. Хэрэв та нэг код, функцийг засахад өөр газарт байгаа код, функц дагаж эвдэрч байвал.

     ##  Төвөгтэй байдал нэмэгдэх ямар ямар шалтгаанууд байдаг вэ?
    ### Жишээ
    
    ## 1. Олон функц эмх замбараагүй холбогдох 

    ### Муу жишээ 
    - Нэг функц хэт олон зүйл хийх 
    - Нэгийг нь өөрчлөхөд бусад нь эвдрэх
    - Аль хэсэг нь ямар үүрэгтэй нь ойлгомжгүй 

    ### Давуу тал 
    - Үүргүүд салсан 
    - Засварлахал амар 
    - Уншихад ойлгомжтой 

    ## 2. Код давхардалт (Code Dupalication)
    
    ### Муу жишээ
    - Ижил код олон газар давхардсан 
    - Хувь өөрчлөгдвөл бүгдийг засна

    ### Давуу тал 
    - Нэг газар л засна
    - Код богино
    - Алдаа багасгана

    ## 3. Хэт олон хариуцлага нэг хэсэгт төвлөрөх 

    ### Муу жишээ
    - Нэг класс хэт олон ажил хийж байна.
    - Өөрчлөлт хийхэд эрсдэлтэй.
    - Туршихад хэцүү.

    ### Давуу тал:
    - Тус бүр нэг үүрэгтэй.
    - Засварлахад амар.
    - Багаар ажиллахад тохиромжтой.

    ## 4.  Нэршил ойлгомжгүй байх

    ### Муу жишээ 
    - Ямар утгатай нь ойлгомжгүй.
    - Шинэ программист ойлгохгүй.

    ### Давуу тал
    - Код өөрөө тайлбар болж өгдөг.
    - Documentation бага шаардана.
